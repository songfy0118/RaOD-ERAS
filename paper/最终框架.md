# 最终框架

## 论文题目

**RaOD-ERAS: Road-Prototype Anomaly Heatmap with Ego-Lane Risk-Aware Refinement for Unexpected Road Obstacle Segmentation**

中文可写：

**面向智能驾驶突发道路障碍的道路原型异常热力图与自车道风险感知分割方法**

## 论文到底做什么

不是做“猫狗小孩分类”，也不是重新训练一个大型检测器。

我们做的是：

**输入一张道路 RGB 图，输出一张异常障碍热力图和一张二值分割 mask，判断道路上哪里可能有未知障碍。**

最终流程：

```text
RGB道路图像
  -> DINOv2预训练视觉特征
  -> 道路原型建模
  -> 初始异常热力图
  -> ERAS自车道风险感知修正
  -> 改进热力图
  -> 二值障碍mask
  -> AP / F1 / IoU / FPR95评价
```

## 读过后采用的主线论文

1. **SegmentMeIfYouCan**
   - 作用：提供 RoadObstacle / ObstacleTrack benchmark。
   - 我们用它的数据集和评价口径。

2. **PEBAL**
   - 作用：强训练式异常分割 baseline。
   - 我们引用它说明现有强方法通常需要训练和较重模型。

3. **Maskomaly**
   - 作用：证明 training-free / mask-level 异常分割是合理方向。
   - 我们借鉴“无需从零训练也可做异常分割”的思路。

4. **Mask2Anomaly**
   - 作用：说明只看像素分数不够，mask/component 级推理可以减少误检。
   - 我们的 ERAS 就是轻量 component-level refinement。

5. **DINOv2**
   - 作用：提供强语义视觉特征。
   - 我们用它生成“道路原型异常热力图”，不是只靠颜色差。

## 我们的方法

### 1. DINOv2 特征提取

用 DINOv2 从 RGB 图像提取 patch 级语义特征：

```text
F = DINOv2(I)
```

### 2. 道路原型建模

用图像下方中间的道路先验区域估计“正常道路特征原型”：

```text
P_road = mean(feature patches inside road-prior region)
```

### 3. 初始异常热力图

每个 patch 与道路原型越不像，异常分数越高：

```text
A(p) = 1 - cosine_similarity(F(p), P_road)
```

得到第一张热力图。

### 4. ERAS 修正

ERAS = **Ego-Lane Risk-Aware Segmentation Refinement**

它做四件事：

1. road-region prior;
2. ego-lane prior;
3. near-field weight;
4. connected-component small-obstacle calibration.

输出：

```text
改进热力图
二值mask
障碍连通域风险排序
```

## 创新点怎么写

安全、合理的创新点：

1. 提出一种 training-free 的智能驾驶道路异常障碍分割框架。
2. 用 DINOv2 语义特征构造道路原型异常热力图，避免只依赖颜色/边缘。
3. 提出 ERAS 自车道风险感知修正模块，结合道路区域、自车道、近场和小障碍连通域信息。
4. 在 SegmentMeIfYouCan RoadObstacle 数据集上做定量和定性验证。

不要写：

```text
SOTA
全新大模型
大规模监督训练
```
